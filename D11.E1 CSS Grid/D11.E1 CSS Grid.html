SECTION 3 (#s3)

Repeat, Auto-Fill, MinMax, and Justify-Content


Note: ‘repeat’ is used to declare repeated columns (or rows). Apart from declaring a fixed number of columns, ‘repeat’ can also be declared with the value ‘auto-fill’, which increases or decreases the number of columns according to the space that's available in the grid container — and subject to the column width that's specified. If there's space for a new column, it gets added to the container.

You'll usually get some extra space in the grid container if you use repeat or autofill with a pixel unit. This can be further handled by either setting justify-content on the grid container (which positions the grid tracks as a whole along the row axis), or by declaring the length unit as minmax value (first unit is minimum, second unit is maximum, usually set to 1fr to ensure it fills the available space).

For this exercise, you'll have to edit both this HTML document and the CSS file named "CSS_TO_EDIT.css" (in the same root folder as this HTML file), making the changes mentioned below.

First, take a look at the broad DOM structure for this document.

There are four top-level <section> elements in this HTML document, tagged with IDs #s1 to #s4. In each section there's an <aside> then an <article>, and followed by an <aside> again.

The <article> elements will be our grid containers for this exercise, and the nested <div> elements within this <article> are going to be the grid items that you're going to style.

General styling

  1. Start by declaring display: grid; on all articles. This will activate the Grid layout for these containers.
  2. Each grid item (the div directly nested within each article) needs centrally-aligned text, a margin of 2px, and a padding of 5px.
  3. Once you're done with the above, observe how the grid-items appear within all the sections.


Section 1 (#s1):

  1. In #s1, the grid container (article) needs 4 columns of equal size. Use the CSS property ‘grid-template-columns’ with a value of ‘repeat(4, 1fr)’to get this — this is equivalent to declaring ‘grid-template-columns: 1fr 1fr 1fr 1fr;’.
  2. Now, go to the HTML markup. Within the second aside element in this section, there are three fieldsets containing radio buttons: one fieldset that toggles between all the ‘justify-items’ property values for the grid container; and one that toggles between all the ‘align-items’ values, and a last one to toggle between all the ‘align-content’ values. Add the correct values to each radio input's ‘value’ attribute, to activate the radio buttons.
  3. In the markup of this section's article, there is an absolutely-positioned, hidden element with the ID ‘grid-guide’, which has its display set to none. Set its display to grid, and give it the same grid-template-columns as the article.
  4. Once you've done the above, go back to the webpage and observe how changing each property (justify-items, align-items, and align-content) affects the grid items within the container. You can also toggle the grid guide for your reference with the checkbox, to see the grid ‘cells’ that are created for each item.
